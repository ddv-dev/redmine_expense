# redmine_expense

Плагин для Redmine: автоматическое списание расходных материалов со склада при работе с задачами, с формированием PDF-актов и простой электронной подписью. У каждого проекта — свой склад и свой список пользователей с доступом к функционалу плагина (не привязано к ролям Redmine).

## Требования

- Redmine 6.0.x
- Ruby (версия, под которую собран сам Redmine — на тестовом сервере это `3.3.6` через rbenv)
- MySQL/MariaDB
- Системный пакет `wkhtmltopdf` (для генерации PDF-актов) — см. раздел ниже, штатный гем не работает на новых версиях Ubuntu

## Установка

Все команды — от пользователя, под которым реально работает Redmine (обычно `redmine`, НЕ `root`; Passenger запускает приложение от этого пользователя, и `bundle install`/миграции нужно делать от него же, иначе будут проблемы с правами на гемы и файлы).

### 1. Скачать плагин

```bash
cd /var/www/redmine/plugins
git clone -b claude/redmine-expense-plugin-xxl1jz https://github.com/ddv-dev/redmine_expense.git
```

Если плагин уже когда-то ставился на этот сервер и его нужно обновить — см. раздел "Обновление" ниже, а не устанавливать заново поверх.

### 2. Поставить гем-зависимости

```bash
cd /var/www/redmine
bundle install
```

Гемы плагина (`roo`, `caxlsx`, `wicked_pdf`, `wkhtmltopdf-binary`) подтягиваются автоматически из `plugins/redmine_expense/Gemfile` — отдельно ничего указывать не нужно, Redmine сам собирает Gemfile-плагинов при `bundle install` в корне.

### 3. Настроить wkhtmltopdf (генерация PDF-актов)

Гем `wkhtmltopdf-binary` тянет за собой готовый бинарник, но это фактически Ruby-скрипт, который сам проверяет версию ОС и отказывается запускаться на неподдерживаемых версиях (например, свежие Ubuntu). Из-за этого надёжнее поставить `wkhtmltopdf` как системный пакет и явно указать путь к нему:

```bash
# Пример: скачать .deb с официальных релизов wkhtmltopdf/packaging под вашу ОС
# https://github.com/wkhtmltopdf/packaging/releases
wget <ссылка-на-подходящий-.deb>
sudo apt install ./wkhtmltox_*.deb

# Проверить, куда встал бинарник
dpkg -L wkhtmltox | grep bin/wkhtmltopdf
```

Дальше указать путь плагину — либо через переменную окружения (например, в конфиге Passenger/systemd для приложения), либо через настройки плагина после установки (поле `wkhtmltopdf_path` в `Setting.plugin_redmine_expense`, задаётся из консоли, см. ниже):

```bash
RAILS_ENV=production bundle exec rails runner "
s = Setting.plugin_redmine_expense
s['wkhtmltopdf_path'] = '/usr/bin/wkhtmltopdf'  # путь из dpkg -L выше
Setting.plugin_redmine_expense = s
"
```

Если используете переменную окружения `WKHTMLTOPDF_PATH` — она имеет приоритет над настройкой из БД.

### 4. Применить миграции

```bash
cd /var/www/redmine
RAILS_ENV=production bundle exec rake redmine:plugins:migrate
```

Важно: `RAILS_ENV=production` для `rake`-задач указывается ПОСЛЕ имени таска (это обычный синтаксис rake, переменные окружения через `key=value` в конце команды). А для `rails runner` — наоборот, ПЕРЕД командой (`RAILS_ENV=production bundle exec rails runner "..."`), иначе Rails по умолчанию загрузит `development`-окружение и упадёт на гемах, которых нет в проде (например `listen`).

### 5. Перезапустить приложение

```bash
touch /var/www/redmine/tmp/restart.txt
```

Redmine регистрирует пункты меню, права и хуки плагина один раз при старте процесса — простого скачивания кода недостаточно, без перезапуска изменения не подхватятся. Если Redmine запущен не через Passenger, а как отдельный systemd-сервис — перезапустите его (`sudo systemctl restart <имя-сервиса>`).

## Обновление плагина (если уже установлен)

```bash
cd /var/www/redmine/plugins/redmine_expense
git pull origin claude/redmine-expense-plugin-xxl1jz

cd /var/www/redmine
bundle install
RAILS_ENV=production bundle exec rake redmine:plugins:migrate
touch tmp/restart.txt
```

Если менялись файлы в `assets/` (JS/CSS плагина), дополнительно:

```bash
RAILS_ENV=production bundle exec rake assets:precompile
```

## Настройка после установки

### 1. Общие настройки плагина (Administration → Plugins → Redmine Expense plugin → Configure)

Здесь задаются правила, общие для всех проектов:
- **Трекеры** — для каких трекеров задач вообще появляется блок добавления материалов.
- **Статус "В работе" / "Решена" / "Закрыта"** — на каких статусах задачи можно добавлять материалы (В работе/Решена) и на каком статусе списание подтверждается автоматически (Закрыта).

Без заполнения статусов "В работе" и "Решена" блок добавления материалов не появится ни в одном проекте.

### 2. Включить плагин для конкретного проекта

В проекте: **Настройки → Модули** → включить галочку **"Расход"**. Без этого плагин в проекте не работает вообще (это обычный модуль Redmine, как "Задачи" или "Wiki").

### 3. Назначить доступ пользователям в проекте

В том же проекте: **Настройки → Расход** (новая вкладка появляется только после включения модуля и только для администраторов проекта — то есть у кого есть право `edit_project`). На вкладке два списка участников проекта с поиском:

- **Доступ к вкладке "Расход" (полный доступ)** — история списаний, склад, промежуточная таблица, импорт, подтверждение/отклонение списаний.
- **Доступ к добавлению материалов в задачу** — может добавлять расходные материалы в карточке задачи. Пользователи из списка выше это тоже могут автоматически.

Доступ выдаётся конкретным пользователям, а не ролям Redmine — список пользователей ограничен участниками именно этого проекта.

### 4. Импортировать остатки склада

В проекте (для менеджеров плагина): вкладка **Расход → Импорт** — загрузка остатков из Excel. У каждого проекта склад свой, импортировать нужно отдельно в каждом.

## Устранение неполадок

- **После git pull изменения не видны** — не забыт ли `touch tmp/restart.txt` (Passenger) или рестарт сервиса.
- **PDF-акт не формируется** — смотреть `log/production.log` на ошибки `wkhtmltopdf`; проверить `wkhtmltopdf_path` в настройках плагина и что бинарник реально исполняемый (`dpkg -L wkhtmltox | grep bin/wkhtmltopdf`, затем `<путь> --version`).
- **Не видно вкладки "Расход" в проекте** — проверить: включён ли модуль в Настройках проекта, назначен ли пользователь менеджером/контрибьютором на вкладке "Расход" в настройках проекта, и был ли перезапуск приложения после установки/обновления плагина.
- **Не видно блока добавления материалов в задаче** — помимо доступа пользователя, задача должна быть в трекере и статусе, разрешённых в общих настройках плагина; также сама задача должна быть редактируемой этим пользователем (в т.ч. с учётом Workflow-прав на поля — например, если в Workflow поле разрешено менять только исполнителю задачи, форма не покажется, если задача не назначена на этого пользователя).
